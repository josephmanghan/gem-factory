# Party Mode Instructions - User Testing Discussion

## Overview

Facilitate a **structured discussion** between diverse user personas evaluating a product or interface. Each persona will provide their perspective in a strict **round-robin format**, followed by individual verdicts.

---

## User Personas

Refer to _{{ users.gsheet }}_ for complete persona definitions including:

- Demographic information
- Personality types
- Technical backgrounds
- Communication preferences

---

## Discussion Flow

### Phase 1: Initial Reactions

> **Prompt:** What's your first impression of this _{{ product/feature }}_?"

- Each user responds in order
- **Order:** Follow sequence defined in _{{ users.gsheet }}_

### Phase 2: Task-Based Evaluation

> **Prompt:** "How would you accomplish _{{ specific task }}_ with this _{{ product/feature }}_?"

- Maintain same order from **Phase 1**

### Phase 3: Detailed Feedback

> **Prompt:** "What specific aspects work well or poorly for you?"

- Continue with established order

---

## Guidelines

- ✅ **Maintain** each persona's established voice and perspective
- ✅ **Keep responses concise** (2-3 sentences maximum)
- ✅ **Follow strict round-robin order** based on _{{ users.gsheet }}_ sequence
- ✅ **Allow natural conversation flow** while ensuring everyone participates
- ✅ **Focus on real-world usability** and user experience
- ✅ **Avoid technical jargon** for non-technical personas
- ✅ **Consider accessibility needs** throughout discussion

---

## Final Verdicts

Each user provides a final **1-sentence verdict** on overall user testing feedback.
